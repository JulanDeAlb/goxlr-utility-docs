Everything around the API development of the GoXLR Utility

📄️ Getting started

The GoXLR Utility is an 'API Driven' application, meaning all configuration happens via an API.

📄️ Unix Socket/Named Pipe

These should be your primary point of entry to the utility, even if you intend to use the HTTP channels for actual work.

📄️ Simple HTTP Request

Once you have the web server details (or assumed the defaults), you can use HTTP to send commands to the daemon.

📄️ Websocket Request

Websockets behave slightly differently than simple requests due to extra async behaviors to consider.
